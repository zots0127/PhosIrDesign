# PhosIrDesign Predictor Backend

This FastAPI service exposes release-generated XGBoost models through a small HTTP API.

## Run Locally

```bash
uvicorn backend.app:app --host 0.0.0.0 --port 8000 --reload
```

Open:

```text
http://127.0.0.1:8000/docs
```

## Docker

Build from the repository root:

```bash
docker build -f backend/Dockerfile -t phosirdesign-api .
docker run --rm -p 8000:8000 phosirdesign-api
```

## Endpoints

| Endpoint | Method | Purpose |
| --- | --- | --- |
| `/health` | GET | Check that models and ligand vocabulary load correctly. |
| `/ligands` | GET | Return known ligands from `data/PhosIrDB.csv` with L1/L2/L3 role counts. |
| `/predict` | POST | Predict emission wavelength and PLQY from selected L1/L2/L3 SMILES. |

## Environment Variables

| Variable | Default |
| --- | --- |
| `PHOSIR_MODEL_DIR` | `backend/model_artifacts` |
| `PHOSIR_DATA_PATH` | `data/PhosIrDB.csv` |
| `PHOSIR_ALLOWED_ORIGINS` | `http://localhost:5173,http://127.0.0.1:5173,https://zots0127.github.io` |

## Use Release Models

`backend/model_artifacts/` is intentionally empty in the source repository. Manual GitHub releases include a full workflow archive and a smaller XGBoost model bundle:

```text
PhosIrDesign-<version>-xgboost_models.tar.gz
```

To run the backend with models generated by a specific release:

```bash
tar -xzf PhosIrDesign-<version>-xgboost_models.tar.gz
PHOSIR_MODEL_DIR="$PWD/xgboost_models" uvicorn backend.app:app --host 0.0.0.0 --port 8000
```

The GitHub Pages predictor does not run Python models in the browser. It calls this API, so using the release-matched model bundle is the recommended way to keep online predictions aligned with a published release.
